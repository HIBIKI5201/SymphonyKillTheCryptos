- AIによるファイル変更の使用はサマリー追加、コメント追加、タイポ修正の編集に留める

## サマリー・コメント

- 全てのメソッドにはサマリーを付ける
- パブリックなプロパティとイベントにはサマリーを付ける
- プロパティ, イベント, フィールド変数かつ一行のサマリーは以下のように一行かつスペースを入れて書く
/// <summary> comment </summary>
- クラス, メソッドまたは二行以上のサマリー以下のように改行してインデントを上げて書く
/// <summary>
///     comment1
///     comment2
/// </summary>
- コメントは基本的に日本語を使用する
- 日本語の最後は「。」で終える
- 語句は「である」調にする
- メソッド内の処理には適宜コメントを付ける
- SerializeField属性を使用するフィールドには、ToolTipAttributeでの説明を付ける

## 命名規則

- C#のフィールド変数はキャメルケースでプレフィックスに `_`
- C#のクラス、プロパティ、メソッドはパスカルケース
- C#の引数はキャメルケース
- 定数はアッパースネークケース
- イベントのプレフィックスは `On`
- イベントハンドラメソッドのプレフィックスは `Handle`
- インターフェースは `I` で始める
- メソッドは動詞で始める
- bool型のプロパティは `Is` または `Has` で始める
- uxml, ussの命名はロウワーチェーンケース

## その他

- `using` ディレクティブはファイルの先頭にまとめて記述する
- フィールド変数を公開する際はプロパティを使用する
- アクセス修飾子は必ず明示的に記述する
- マジックナンバーは排除し、代わりに定数で定義する
